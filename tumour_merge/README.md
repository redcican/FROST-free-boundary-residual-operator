# FROST benchmark — two-tumour merge (topology-change free-boundary problem)

The headline FROST benchmark: two tumours grow and **coalesce** (connected components 2 → 1). This
is the case FBNO's single diffeomorphism *cannot* represent (no topology change) and FROST's
level-set-on-a-fixed-grid handles natively.

**Generator: `gen_tumour_merge.py`** (v2, current) → `tumour_merge.npy` (+ summary, preview).
Use `python viz_tumour_merge_2d.py` for the flat preview and `python viz_tumour_merge_3d.py` for
the FBNO-style 3D figure. CPU, fast. **Run single-threaded**
(`OPENBLAS_NUM_THREADS=1 OMP_NUM_THREADS=1 MKL_NUM_THREADS=1`) — the sparse solves spawn 22 BLAS
threads otherwise.

> **v1 vs v2.** v1 (`gen_tumour_merge.py`) grew smooth *disks* → smooth tubes — **oversimplified**,
> losing the organic morphology of the real FBNO tumours. **v2 (`gen_tumour_merge.py`) keeps the
> original topology/morphology**: each tumour is the actual FBNO geometry-A ground-truth contour
> sequence from `domain_test.npy` / `reproduce_panel_a_3d.py`. Samples use A+A pairs, preserving the
> original contour orientation. The two placed contour sequences are combined with a continuous
> polygon union at every time step, producing two boundary components before contact and a new single
> exterior boundary after contact. The union is then rasterised into FROST level sets.

## Why we built it (vs reusing FBNO's tumour data)
FBNO's tumour data is **single-topology** (one growing domain) and was generated by **MATLAB
FEM/FVM** code (`gopikrishnancr/2D_tumour_growth_FEM_FVM`) that is not in the repo and not runnable
here. A merge benchmark needs a *new* generator, and a **level-set** generator is the natural choice
— it is exactly FROST's geometry representation and merges automatically.

## Model (v2 — FBNO geometry-A topology, polygon-union merge)
Fixed box `D = [-1,1]²` (128×128), domain `Ω_t = {φ<0}`, `φ` a signed-distance level set:
- **geometry**: FBNO geometry A is loaded from `domain_test.npy`, recentered per time slice, and
  scaled once globally per sample. This preserves the original relative growth and contour topology
  instead of forcing each frame to a constant radius.
- **new merged topology**: two geometry-A source contour sequences are placed at `(±sep(t),0)`
  without random rotation/mirroring, and combined with `shapely.unary_union`. Before contact the continuous geometry has two polygon
  exteriors; after contact it has one newly created exterior boundary. Candidates are accepted only
  when the continuous polygon topology, raster mask topology, and stored boundary components agree
  for every time step and no mask touches the box edge.
- **level-set ground truth**: the polygon union is rasterised on the fixed FROST grid, then
  `φ = signed_distance(mask)` by EDT. This is the geometry FROST learns and the topology change is
  represented directly on the fixed grid.
- **nutrient** `u` re-solved on the merged domain: `(-Δ + k)u = 0`, `u=1` on `Γ` → high at the rim,
  lower in the core (as in FBNO Fig. 4).

Per sample, varied: global contour scale, `sep0`, `sep1`, and `k∈[1,5]`. The 3D view
(`viz_tumour_merge_3d.py`) is a 1×5 row matching the flat preview's selected times: `t=0`, pre-merge,
merge, post-merge, and final. Each panel renders the cumulative z=time free-boundary surface in the
same style as FBNO's `panel_a_3d_combined.png`, including the same orthographic view, `z=0..100`
time scale, `plot_trisurf` settings, YlGnBu radial-distance colouring, red `t=0` caps, and stacked
boundary rings.

## Format (`tumour_merge.npy`, object array of dicts)
Each sample:
- `phi (15,128,128) float32` — signed-distance level set, negative inside Ω.
- `u (15,128,128) float32` — nutrient field, zero outside Ω.
- `mask (15,128,128) bool` — raster tumour domain.
- `boundary_components (15,) object` — list of resampled polygon exterior arrays per time step
  (`2` arrays before merge, `1` array after merge).
- `source_indices (2,) int32` — FBNO geometry-A `domain_test.npy` source contour indices.
- `params [gA,gB,global_scale,sep0,sep1,rotA,rotB,k]`.
- `n_components (15,)` and `merge_step` — the validated 2→1 topology labels.

## Validation & visualization
80/80 samples merge, merge steps currently span t=10–12 (see `tumour_merge_summary.json`).
- `tumour_merge_preview.png` — a flat 1×5 counterpart to the 3D scenario, using the same selected
  panels, FBNO geometry-A radius color scale, and pre/post-merge display logic.
- `tumour_merge_3d.png` (`python viz_tumour_merge_3d.py`) — a **1×5 FBNO-style 3D space-time
  view** of the same selected times as the flat preview. It starts at `t=0`, then shows cumulative
  surfaces through pre-merge, merge, post-merge, and final time, with z as time and the exact stored
  boundary Γ overlaid.

## How FROST uses it
- **C1 forward operator (few-shot):** train on a handful of samples, predict `(u, φ/Γ)` evolution
  on held-out samples — **report field L² + interface IoU/Hausdorff *through the merge event***.
- **Headline comparison:** FBNO's diffeomorphism baseline must fail at the merge (genus change);
  FROST's level-set should track it. `n_components` / `merge_step` label the event for per-event
  metrics.
